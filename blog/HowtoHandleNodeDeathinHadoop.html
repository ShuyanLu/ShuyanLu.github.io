How to handle node death in hadoop?

Posted on 03/14/2015 to Blog | Tags: hadoop

My suggestion is running jps twice after start-all.sh cause I occasionally find node dying right after the first jps command.

If you find any nodes dying, here is the common process to handle it.

$ cd $HADOOP_HOME
$ sbin/yarn-daemon.sh stop nodemanager
$ sbin/hadoop-daemon.sh stop datanode
$ sbin/hadoop-daemon.sh start datanode && /usr/local/hadoop/sbin/yarn-daemon.sh start nodemanager

This process shows the common way to handle node death, but it may needs a little tweak depending on which node is killed. And always be cautious of formatting namenode.
